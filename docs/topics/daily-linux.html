<body> <header>Linux日常</header> <main> <section> <p><strong>创建于 2022-7-20 15:50</strong></p> <p>该专题是用来记录一些在<b>Linux</b>上的所有事情, 包括使用的软件, 遇到的问题以及其解决方案.</p> <p>创建这个专题的原因有两个:</p>

<p>1. 最近买了新的笔记本 HP Elitebook 865 G9, 对 Linux 的兼容非常不错(本人用的 Ubuntu 22.04 LTS, 请各位大佬不要嘲讽我这只菜鸡)</p> <p>算是找到了自从16年入坑 Linux 以来心心念念的梦中情本了</p> <p>有着8到10个小时的续航(暴打英特尔12代的笔记本), GTX 1050 TI 水平的核显, 以及兼容雷电3并且传输速率接近英特尔的雷电4 的 <strong>USB4</strong>(是的, 现在 AMD 的笔记本也可以外接显卡了).</p>

<p>2. 使用了这么长时间的 Linux 还是停留在用来写代码的阶段上, 当时选 Ubuntu 的原因就是想有一个开箱即用的环境.</p> <p>当长期处于一个问题的环境中, 加上不把遇到使用时遇到的问题和解决方案记录下来, 时间一长再遇到同样的问题时你就会现以前的白折腾了, 因为你一点成长都没有.</p> <p>当然我以前不做这些记录是因为很多都是配置环境的问题, 基本上都是有时效性的问题, 比如你这个东西在 Ubuntu 18 上是这么弄的, 而到了 Ubuntu 22 上这个东西的结构变了, 这个解决方法就不生效了.</p> <p>不过时间证明了我这个想法是错误的, 因为在解决问题的过程中是有一种看不见的东西叫做思路, 的确问题和解决方法是有时效性的, 思考问题的思路是没有的.</p> <p>所以我想现在开始记录这些问题, 并且系统的学习 Linux 的使用.</p>
